Notes on Integration Testing Approach

  • All tests hit a real PostgreSQL 16 database — no mocking ensures ORM, middleware, and response shapes are production-accurate.
  • TC-C03 accepts 201 or 409 defensively: if a previous teardown failed, the existing application record prevents a duplicate apply.
  • Planned tests follow the same pattern: beforeAll seeds data, afterAll cleans it up with cascade.
  • Password hashing uses bcrypt cost factor 12 in both test setup and the production auth service — matching environments prevents hash divergence bugs.
