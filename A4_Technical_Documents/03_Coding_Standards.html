1. Style Guide Adopted

The CreConnect project follows the Airbnb JavaScript Style Guide — one of the most widely adopted standards in the JavaScript / Node.js ecosystem. ESLint 8 enforces these rules automatically across the backend codebase. The frontend (React 18 / Vite) additionally uses eslint-plugin-react and eslint-plugin-react-hooks.

Run the linter

cd creconnect-backend
npm run lint

4. Functions & Async Patterns

4.1 Async/Await (preferred over .then().catch())

// ✅ Correct — async/await with try/catch
const getUser = async (id) => {
  const user = await User.findByPk(id);
  return user;
};

// ❌ Avoid — callback chaining
User.findByPk(id).then(user => ...).catch(err => ...);

4.2 Controller Pattern (all routes)

// Standard across all 12 controllers
const createCampaign = async (req, res, next) => {
  try {
    const campaign = await campaignService.create(req.body, req.user.id);
    return res.status(201).json({ success: true, data: campaign });
  } catch (err) {
    next(err); // passed to global error handler
  }
};

7. Destructuring

// ✅ Always destructure from req.body and model includes
const { email, password, role } = req.body;

const { id: userId, brandProfile } = await User.findByPk(req.user.id, {
  include: [{ model: BrandProfile, as: 'brandProfile' }]
});

8. React / Frontend Standards

AreaRule
Component typeFunctional components only — no class components
State managementuseState, useReducer, useContext — no legacy lifecycle methods
Side effectsuseEffect with explicit dependency arrays
API callsCentralised in src/api/ or src/services/ — never inline in components
List rendering keysStable unique keys (item.id) — never array index
PropsDestructured directly in function signature, validated with prop-types

Correct Component Pattern

// ✅ Correct — functional, destructured props, conditional render
const CampaignCard = ({ campaign, onApply }) => {
  const { title, niche, budgetPKR, status } = campaign;

  return (
    <div className="campaign-card">
      <h3>{title}</h3>
      <span>{niche} — PKR {budgetPKR.toLocaleString()}</span>
      {status === 'PUBLISHED' && (
        <button onClick={() => onApply(campaign.id)}>Apply</button>
      )}
    </div>
  );
};

export default CampaignCard;

// Get the user by id
const user = await User.findByPk(id);
